What is an Integrity Checksum?

A growing number of library providers are generating HTML snippet for their resources with a Subresource Integrity checksum. This helps to protect the users of their libraries as compliant browsers will not execute a resource with an integrity value unless the checksum of the response matches the value provided.

How do I find the Integrity Checksum?

On the site where you found the URL for your library, there was most likely a section with HTML snippets that you could copy/paste into your HTML to include the library in your project. In this snippet, there will be an integrity attribute if the library provider supports Subresource Integrity checks.

For a Javascript resource, the snippet would look like:
<script src="<dependency-url>" integrity="<hashing-algorithm>-<base-64-encoding-checksum>"></script>

For a Stylesheet resource, the snippet would look like:
<link href="<dependency-url>" rel="stylesheet" type="text/css" integrity="<hashing-algorithm>-<base-64-encoded-checksum>" />

What do I do if no Integrity Checksum is provided in the snippet?

In cases where no Integrity Checksum is provided by the library provider, you can leave the Integrity Checksum field blank. Once the dependency file is retrieved, Blueprint will generate a SHA-384 checksum that can be used to verify the integrity of the library in all future requests.
